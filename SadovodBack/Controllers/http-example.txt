Контроллер ValueController
Атрибут главного [Route("api/[controller]")] - для работы с данным контроллером. Вместо [controller] в путь пишем название контроллера(...api/Example/...)
Методы:
1)public async Task<JsonResult> Get()
	Атрибуты:
		1.Атрибут пути [Route("GetAll/")] - устанавливает путь для данного запроса
		2.Атрибут действия [HttpGet] - устанавливает тип действия, требуемый сервером для выполнения данного метода
	Метод является асинхронным, возвращает все строки базы данных
2)public async Task<JsonResult> Get(int id)
	Атрибуты:
		1.Атрибут пути [Route("GetByID/")] - устанавливает путь для данного запроса(.../api/Example/GetByID?id=needId - требуемый вид пути)
		2.Атрибут действия [HttpGet] - устанавливает тип действия, требуемый сервером для выполнения данного метода
	Метод является асинхронным, возвращает строку с соответствующим id
3)public async void Post([FromBody] string value)
	Атрибуты:
		1.Атрибут пути [Route("PostNew/")] - устанавливает путь для данного запроса
		2.Атрибут действия [HttpPost] - устанавливает тип действия, требуемый сервером для выполнения данного метода
		3.Атрибут [FromBody], указывающий, что информация будет браться из тела запроса
	Метод является асинхронным, добавляет в базу данных новую запись с новым id
4)public async void Put(int id, [FromBody] string value)
	Атрибуты:
		1.Атрибут пути [Route("PutInID/")] - устанавливает путь для данного запроса(вид пути аналогично GetByID)
		2.Атрибут действия [HttpPut] - устанавливает тип действия, требуемый сервером для выполнения данного метода
		3.Атрибут [FromBody], указывающий, что информация будет браться из тела запроса
	Метод является асинхронным, изменяет строку в базе данных с указанным id
5)public async void Delete(int id)
	Атрибуты:
		1.Атрибут пути [Route("DeleteByID/")] - устанавливает путь для данного запроса(вид пути аналогично GetByID)
		2.Атрибут действия [HttpDelete] - устанавливает тип действия, требуемый сервером для выполнения данного метода
	Метод является асинхронным, удаляет строку в базе данных с указанным id
